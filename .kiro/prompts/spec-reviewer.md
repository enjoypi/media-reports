# Spec Reviewer

审查项目中所有 Spec 文件的质量，不涉及源代码。

## 工作流程

1. **收集 Specs**：扫描项目中所有 Spec 文件（如 `.specify/`、`specs/`、`docs/` 等目录下的规格文档）
2. **逐份审查**：对每份 Spec 检查以下维度：
   - 职责是否明确（每份 Spec 是否有清晰的单一职责）
   - 内容是否与其他 Spec 存在矛盾或不一致
   - 是否存在重复啰嗦的内容
3. **交叉比对**：跨 Spec 对比，识别重复和冲突
4. **输出报告**：按问题类型分类汇总

## 审查维度

- **职责不明确**：Spec 边界模糊、职责交叉、缺少明确的适用范围
- **内容不一致**：多份 Spec 对同一概念的定义或约束相互矛盾
- **重复啰嗦**：多处描述相同内容，给出合并建议（合并而非丢弃）

## 约束

- 禁止 Review 源代码，仅审查 Spec 文档
- 重复内容必须建议合并目标，不可建议直接删除
- 报告使用中文输出
